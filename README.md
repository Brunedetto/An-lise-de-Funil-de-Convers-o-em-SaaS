# Anlise-de-Funil-de-Conversão-em-SaaS

-Contexto

Uma empresa SaaS apresenta um alto volume de usuários cadastrados, porém enfrenta baixa conversão para clientes pagantes.

-Problema de Negócio

Apesar da boa aquisição de usuários, a empresa não consegue converter esses usuários em clientes pagantes, indicando possíveis falhas na jornada do usuário.

-Objetivo

Analisar o comportamento dos usuários ao longo do funil de conversão para identificar gargalos e propor ações que aumentem a conversão.

-Hipóteses
Alto abandono após o primeiro uso
Usuários não encontram valor no produto
Baixo engajamento impacta diretamente a conversão
O plano gratuito pode reduzir o incentivo para upgrade

-Estrutura do Funil
Cadastro: 10.000 usuários (100%)
Ativação: 8.000 usuários (80%)
Engajamento: ~1.000 usuários (~12,5%)
Conversão: 100 usuários (1%)

Usuários engajados foram definidos como aqueles com 3 ou mais acessos à plataforma.

-Principais Análises
Distribuição de acessos dos usuários
Comparação entre usuários engajados e não engajados
Relação entre frequência de uso e conversão
Análise do comportamento dos usuários pagantes

-Insights
A maior perda ocorre na etapa de engajamento
A maioria dos usuários realiza apenas 1 ou 2 acessos
Usuários engajados têm maior probabilidade de conversão
O problema está na retenção, não na aquisição

-Recomendações
Melhorar o onboarding para destacar o valor do produto
Guiar o usuário até o “momento de valor”
Criar estratégias para aumentar o uso recorrente
Revisar o equilíbrio entre plano gratuito e pago

-Conclusão
A análise mostra que o principal desafio está no engajamento dos usuários.
Aumentar o uso recorrente do produto é a principal alavanca para melhorar a conversão e crescimento da receita.

-Ferramentas Utilizadas
Python (Pandas, NumPy, Matplotlib)
